# Google Colab 対応 要件定義

## 1. 背景・目的
- 既存のローカル実行プログラムを、Google Colab 上で誰でも実行できる形にする。
- 利用者は毎月の Excel、PDF、Gemini API キーを設定するだけで処理を実行できるようにする。
- 処理結果は Google Drive に保存し、継続運用できる形にする。

## 2. 現状整理
- 現行プログラムはローカル実行前提である。
- 入力は `src/input` 配下、出力は `src/output` 配下を前提としている。
- Excel 分析処理は `.env` の `GOOGLE_API_KEY` を読み込む実装になっている。
- `分類.csv` は `input` 配下に毎回配置する前提になっている。

## 3. 対応方針
- 実行環境を Google Colab に変更する。
- 入力ファイルは Colab 実行時に利用者が指定できるようにする。GoogleDriveにフォルダパスを指定
- API キーは実行時に入力された値のみを使う。
- `分類.csv` は Google Drive 上の固定パスに配置し、毎回そこを参照する。
- 出力は Google Drive 配下の指定フォルダへ保存する。
- 既存の PDF 分割と Excel 分析ロジックはできるだけ再利用し、入出力と設定方法を Colab 向けに変更する。

## 4. スコープ
### 4.1 対象
- Google Colab からの実行
- 月次 Excel の分析
- PDF の分割
- Google Drive への出力保存
- `分類.csv` の固定参照
- Gemini API キーの実行時入力
- 実行ログをColab上に出力

### 4.2 対象外
- 業務ロジック自体の大幅な変更
- Gemini プロンプト内容の全面見直し
- Web アプリ化
- 複数ユーザー向けの認証管理機能

## 5. 利用シナリオ
1. 利用者が Google Colab ノートブックを開く。
2. Google Drive をマウントする。
3. 利用者が当月の Excel と PDF をアップロードする。
4. 利用者が Gemini API キーをその場で入力する。
5. プログラムが Google Drive 上の `分類.csv` を読み込む。
6. PDF 分割と Excel 分析を実行する。
7. 結果ファイルを Google Drive に保存する。
8. 利用者が Colab 上で実行結果と保存先を確認する。

## 6. 機能要件
### 6.1 実行環境
- Google Colab 上で実行できること。
- Colab 上で必要ライブラリをインストールできること。
- Google Drive をマウントして利用できること。

### 6.2 入力
- 毎回必要な入力は以下とする。
- 月次 Excel ファイル（`.xlsx`）
- PDF ファイル（`.pdf`）
- Gemini API キー
- Excel と PDF の指定方法をアップロードにする

### 6.3 API キー制御
- API キーは必ず実行時に利用者が入力した値を使うこと。
- `.env` やソースコードに定義済みの API キーは使わないこと。
- 実行環境に既存の `GOOGLE_API_KEY` が存在していても、それを自動採用しないこと。（Colabの環境変数を使わない）
- API キーをファイルへ保存しないこと。
- API キーをログへ全文出力しないこと。

### 6.4 `分類.csv` 管理
- `分類.csv` は毎回アップロード不要とする。
- Google Drive 上の固定パスに保存し、プログラムは毎回その場所を参照すること。
- 担当者が Google Drive 上で直接編集または差し替えできること。
- 実行時は常に最新の `分類.csv` を読み込むこと。

### 6.5 出力
- 出力先は Google Drive とする。
- PDF 分割結果を Google Drive に保存すること。
- Excel 分析結果を Google Drive に保存すること。
- 実行結果ごとに出力フォルダを分けられること。
- 利用者が保存先パスを確認できること。

### 6.6 実行制御
- PDF 分割のみ、Excel 分析のみ、両方実行のいずれも可能にすること。
- 入力ファイルが不足している場合は、理由を明示して停止または該当処理のみスキップすること。
- 実行結果サマリを Colab 上に表示すること。

### 6.7 ログ・エラー
- どのファイルを処理したか Colab 上で確認できること。
- エラー発生時に対象ファイルと理由を確認できること。
- Excel 分析エラーは従来どおり出力ファイルにも残せること。

## 7. 非機能要件
### 7.1 操作性
- 利用者（ITに詳しくない人）が Colab ノートブックの手順に従えば実行できること。
- 毎月の作業は「ファイル指定」「API キー入力」「実行」の最小手順にすること。

### 7.2 保守性
- 既存ロジックを流用しつつ、Colab 固有の処理を分離すること。
- ローカル実行ロジックと Colab 実行ロジックの責務を明確にすること。

### 7.3 セキュリティ
- API キーの永続保存を行わないこと。
- API キーを既存設定から自動取得しないこと。
- Drive 上の共有ファイルは必要最小限とすること。

## 8. 想定構成
### 8.1 ファイル配置の考え方
- Python の処理本体は既存 `src` 配下をベースに継続利用する。
- Google Colab 用の実行入口を別途追加する。
- `分類.csv` は Google Drive 上の固定ディレクトリに保存する。
- 出力は Google Drive 上の出力ディレクトリに保存する。

### 8.2 推奨ディレクトリ例
- 入力一時領域: Colab セッション上の一時ディレクトリ
- 分類マスタ: `MyDrive/businessDX/setting/分類.csv`
- 出力先: `MyDrive/businessDX/Output/{実行年月または実行日時}/`

## 9. 改修要件
### 9.1 必須改修
- `.env` 読み込み依存を撤廃または無効化する。
- Excel 分析処理に API キーを引数で渡せるようにする。
- `分類.csv` の参照先を引数化または設定化する。
- 出力先を Google Drive パスで指定できるようにする。
- Colab から実行するためのノートブックを追加する。

### 9.2 推奨改修
- 実行設定をまとめる設定オブジェクトまたは設定関数を追加する。
- 実行前チェックを追加し、入力不足や設定ミスを早期に検知できるようにする。
- Colab 用 README または利用手順書を追加する。

## 10. 受け入れ条件
- Google Colab 上で依存関係のセットアップ後に実行できること。
- 利用者が入力した API キーで Gemini 呼び出しが行われること。
- `.env` に API キーが存在しても、それが使われないこと。
- `分類.csv` を毎回アップロードしなくても実行できること。
- `分類.csv` を Drive 上で更新した内容が次回実行に反映されること。
- PDF 分割結果が Google Drive に保存されること。
- Excel 分析結果が Google Drive に保存されること。
- 実行結果サマリとエラー情報を Colab 上で確認できること。
